## 填写说明

### 1. 基本信息的填写
- **剧集编号**：按照目录文件中的编号填写（例如：Episode-01）
- **剧集标题**：使用 catalog.md 中对应的标题
- **目标时长**：根据分镜数量计算（每集建议30-60秒，每镜头5-10秒）
- **视频视角**：根据故事需要选择，明确一种
- **视觉风格**：根据项目整体风格选择

### 2. 主角外观描述的填写（关键）

**这是确保跨镜头角色一致性的核心！**

请尽可能详细描述主角的外观特征，因为每个分镜的提示词都需要完整复制这一段描述。

**示例**：
```
主角姓名：林晚
外观特征：
- 年龄：28岁
- 性别：女性
- 身高/体型：165cm，纤细但结实
- 发型发色：黑色长发，通常扎成高马尾
- 服装穿搭：灰色连帽卫衣、深色牛仔裤、运动鞋
- 面部特征：瓜子脸，杏仁眼，肤色偏白
- 其他显著特征：右手腕上有一条银色手链
```

**为什么必须重复描述？**
因为AI生图容易出现"角色漂移"现象，每个镜头都重复完整描述，可以最大程度确保生成的角色是同一个人。

### 3. 分镜脚本表格填写指南

#### A. "镜头"列
- 按顺序编号：1, 2, 3, 4...
- 每新增一个镜头，编号递增

#### B. "时长"列（审查重点）

**严格遵守规则：每个镜头 5-10 秒**

填写示例：
- ✅ 正确：5、6、7、8、9、10（均≤10秒）
- ❌ 错误：12、15、20（超出10秒限制）

**如何确定时长？**
1. 分析场景动作的复杂度
2. 简单的表情变化：5-6秒
3. 中等动作（拿起物品、转身）：7-8秒
4. 复杂动作（对话+动作）：9-10秒
5. 绝不能超过10秒！

**⚠️ 强制检查**：填写完成后，必须统计所有镜头时长总和，确保每个镜头都≤10秒

#### C. "场景描述"列

**编写要求**：
- 清晰描述画面内容（人物、动作、环境）
- 注明是否有对话或情绪表达
- 尽量简洁明了（这是给分镜师看的描述）

**示例**：
> 主角推开仓库门，看到整齐堆放的物资，脸上露出惊喜表情

#### D. "文生图提示词"列（关键）

**结构**：
```
{景别选择}，{角色外观}，{主体描述}，{环境场景}，{情绪氛围}，{视觉风格}，{光影效果}
```

**填写要点**：

1. **景别选择（根据视角调整）**
   - **自拍视角**："自拍角度"、"手机前置摄像头视角"、"近距离自拍"
   - **第一人称**："第一人称视角"、"POV"、"主观镜头"
   - **第三人称**："远景"、"中景"、"近景"、"特写"

2. **角色外观（必须完整重复）**
   - 将"主角外观描述"部分完整复制到此处
   - **不要省略任何细节！**

3. **主体描述**
   - 主角在做什么动作（静态瞬间）
   - 表情、姿势、手势

4. **环境场景**
   - 背景、道具、空间布局
   - 确保与前后镜头的空间一致性

5. **情绪氛围**
   - 用关键词描述情绪（惊喜、愤怒、得意、恐惧等）

6. **视觉风格**
   - 与基本信息中的视觉风格一致

7. **光影效果**
   - 具体光源描述（窗户光、灯光、自然光）
   - 光线质量（柔和、硬朗、明暗对比）

**完整示例（第三人称视角）**：
```
中景，28岁女性，黑色长发扎成高马尾，穿着灰色卫衣和牛仔裤，侧身站立，右手推开木制大门，表情平静中带期待，背景是昏暗的仓库内部和整齐的货架，紧张的氛围中透出一丝希望，写实风格，顶部的昏暗灯光投射
```

**完整示例（自拍视角）**：
```
自拍角度，28岁女性，黑色长发扎成高马尾，穿着灰色卫衣和牛仔裤，近距离拍摄，主角手持手机，眼睛看向屏幕，背后是混乱的客厅和散落的物品，疲惫但坚定的情绪，写实风格，手机屏幕的冷光照在脸上
```

#### E. "图生视频提示词"列（关键）

**结构**：
```
{镜头描述}，{镜头运动}，{主体行为}，{背景变化}，{环境变化}
```

**填写要点**：

1. **镜头描述**
   - 概括镜头的主要内容

2. **镜头运动**
   - 推轨（向前/向后）
   - 拉轨
   - 摇摄（左右/上下/360度）
   - 静止
   - 手持晃动

3. **主体行为**
   - 主角的具体动作
   - 从第一帧开始，到动作完成
   - **时长必须在5-10秒内完成**

4. **背景变化**
   - 背景是否移动
   - 道具是否变化

5. **环境变化**
   - 光线是否变化
   - 天气是否变化
   - 特殊效果

**核心技巧**：
- 使用"→"、"立刻"、"紧接着"串联动作顺序
- 结尾必须包含稳定性要求："保持角色面部清晰、光线稳定、无闪烁"

**完整示例**：
```
镜头缓慢推近，主角右手推开大门，随即迈步进入仓库，眼睛扫视四周，脸上从平静转为惊喜，背景货架静止，环境光保持稳定 → 保持角色面部清晰、光线稳定、无色彩闪烁
```

#### F. "备注"列

**用于填写**：
- 特殊注意事项
- 连续多个镜头的关联关系
- 道具的一致性要求
- 音效或配乐建议
- AI生成时的特殊参数

**示例**：
```
本镜头与镜头2在同一空间，注意矿泉水箱位置保持一致
```

## 格式检查清单

完成表格填写后，请使用此清单逐项检查：

### 基本信息检查
- [ ] 剧集编号是否正确
- [ ] 剧集标题与目录一致
- [ ] 视角选择明确（只选一种）
- [ ] 视觉风格明确

### 主角外观检查
- [ ] 外观描述详细完整
- [ ] 包含所有视觉特征（后期每镜头复制）

### 分镜表格检查（关键）
- [ ] 每个镜头都填写了
- [ ] 每个镜头时长在5-10秒之间（无例外）
- [ ] 所有镜头时长总和等于目标时长
- [ ] 场景描述清晰
- [ ] 文生图提示词包含完整角色外观描述
- [ ] 文生图提示词符合格式（景别+角色+主体+环境+氛围+风格+光影）
- [ ] 图生视频提示词结尾有稳定性要求
- [ ] 提示词使用中文编写

### 逻辑一致性检查
- [ ] 跨镜头的角色外观描述一致
- [ ] 同一空间的物品位置一致
- [ ] 相邻镜头组切换了空间（避免冲突）
- [ ] 动作链逻辑清晰（动作可流畅衔接）

## 常见场景的分镜建议

### 单人物场景（独处/行动）

**设计思路**：通过镜头切换打破单调感

**推荐分镜组合**：
1. 全景/中景（交代环境） → 2. 近景/特写（突出动作或情绪） → 3. 中景/特写（收束）

### 对话场景（双人物）

**设计思路**：正反打镜头切换

**推荐分镜组合**：
1. A人物中景 + B人物背影 → 2. B人物中景 + A人物背影 → 3. 双人中景（关系）

### 复仇打脸场景

**设计思路**：强化爽感的对比

**推荐分镜组合**：
1. 反派近景（狼狈） → 2. 主角近景（得意） → 3. 主角近景（立威）

### 囤货展示场景

**设计思路**：突出数量感和满足感

**推荐分镜组合**：
1. 全景（仓库全貌） → 2. 近景/特写（重点物品） → 3. 主角中近景（满意度）

## 模板使用说明

### 使用场景
- 针对每集漫剧（Episode-01 至 Episode-60）
- 可以为每集单独创建一个分镜脚本文件
- 文件名格式：`storyboard-Episode-XX.md`

### 推荐工作流程

#### 第一步：生成全文
- 执行 `/write` 命令完成正文创作
- 获得 Episode-XX.md 文件（含详细剧情描述）

#### 第二步：提取关键场景
- 从 Episode-XX.md 中提取需要视觉化的场景
- 每集可制作1-3个分镜脚本（根据重要程度）

#### 第三步：填写分镜模板
- 创建本文件并填写基本信息
- 详细描述主角外观（完整复制到每个分镜）
- 逐个填写分镜表格（严格遵守时长限制）

#### 第四步：质量检查
- 使用"格式检查清单"逐项核对
- 重点检查时长期（≤10秒规则）
- 确认视角和角色一致性

#### 第五步：AI生成
- 将文生图提示词逐一复制到AI绘画工具（如Midjourney、Stable Diffusion）
- 将图生视频提示词逐一复制到AI视频工具（如Runway、Pika）
- 注意保留首帧图片，作为视频生成的基础

### 进阶技巧

#### 大批量生成策略
- 使用脚本批量读取本模板的表格内容
- 自动发送到AI生成工具API
- 按"镜头"编号保存生成的图片和视频文件

#### 连续剧集的一致性维护
- 创建 "角色外观参考文档"（包含详细描述+参考图）
- 每集分镜脚本都引用该文档
- 定期维护角色外观的一致性
