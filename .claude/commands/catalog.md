# catalog - 规划章节目录

**命令说明**：根据大纲和人物小传，创建详细的章节目录，规划每一章的核心情节、情感变化和结构布局。

**使用场景**：大纲和人物确定后，需要细化每个章节的具体内容，为后续的章节写作提供清晰的蓝图。

## 前置检查

执行本命令前，必须确保：
- `outline.md` 存在（故事大纲）
- `characters.md` 存在（人物小传）

如果任一文件缺失，提示用户先完成前序步骤。

## 工作流程

1. **读取背景资料**
   - 读取 `outline.md`（获取五章结构的核心设定）
   - 读取 `characters.md`（了解人物性格和行为逻辑）
   - 加载 romance-novel-skill 的资源文件：
     - `templates/chapter-index-template.md`（章节目录格式）
     - `examples/catalog-example.md`（优秀章节目录示例）

2. **五章内容规划**

   为每个章节详细规划以下内容：

   - **第一章**：相遇与初印象（起）
     - 开场场景（500字）：具体设定场景、时间、氛围、当事人的状态
     - 男女主相遇（800字）：精确规划相遇方式、初印象、初次对话和冲突点
     - 冲突延续（500字）：后续互动细节、两人的内心波动
     - 章末转折（200字）：留下的悬念或情感冲击
     - 情感变化：女主和男主各自的情感起点和终点
     - 经典金句：能概括本章主题的一句话

   - **第二章**：关系深化与矛盾显现（承）
     - 导致男女双方更多交集的关键事件
     - 展现双方性格魅力的具体情境
     - 被人关注的情感萌芽时刻
     - 潜在的矛盾预兆和障碍显现

   - **第三章**：情感升温与冲突爆发（转）
     - 关系质变的关键时刻
     - 最具甜蜜感的互动细节
     - 强烈冲突的导火索和升级过程
     - 情感跌入谷底的破裂场景

   - **第四章**：分离与反思（合前）
     - 分开后的情感痛苦与成长
     - 双方对关系的重新审视与领悟
     - 误会逐步解开的过程
     - 再次相遇的契机和场景

   - **第五章**：和解与圆满（合）
     - 勇敢追回的决心和行动
     - 深度沟通解开所有心结
     - 情感升华的动人时刻
     - 美好结局的圆满收尾

3. **结构设计**
   - 确保每章字数控制在 2000-3000 字
   - 合理安排小高潮的节奏（每章至少 2-3 个情节点）
   - 通过情感曲线图表展现故事的张力变化
   - 设置前后呼应的细节

4. **输出与保存**
   - 将完整的章节目录保存到 `catalog.md` 文件
   - 向用户确认目录已生成，并简要介绍每章的核心看点
   - 询问用户是否需要调整某些章节内容

## 输出格式

生成的 `catalog.md` 文件包含：
- 整体结构说明（五章设计、字数规划）
- 每章详细规划（情节、情感变化、金句、伏笔）
- 结构检查清单（节奏控制、情感曲线、冲突分布）
- 创作提示（衔接要点、避免事项、加分项）

## 创作要点

- **符合大纲和人设**：所有情节必须基于大纲设定，人物行为必须符合角色性格
- **情节具体化**：不要使用"之后两人关系变好"这类模糊描述，要具体到事件和行动
- **情感递进**：每章的情感变化要清晰，有层次感，不能突兀
- **小高潮设计**：每章都要有吸引读者的情节点，避免平铺直叙
- **前后呼应**：埋设的伏笔要在后续章节有所回应，形成闭环

## 注意事项

- 章节目录一旦确定，将成为后续章节正文创作的主要依据
- 每章的字数分配要合理，不要把重头戏全堆在后期
- 情感冲突要有递进，第三章的高潮前应该有充足的铺垫
- 分离期的痛苦要真实，和解的过程要充分，避免仓促
- 结局要利落圆满，不要拖沓，给读者充分的情感满足

## 下一步

章节目录完成后，通常继续：
1. 撰写具体章节（使用 `/write 章节号` 命令，如 `/write 1`）
2. 从第一章开始逐步完成整篇小说
