# outline - 创建现代言情故事大纲

**命令说明**：根据用户的灵感或想法，创建一个完整的现代言情小说故事大纲。

**使用场景**：用户已经有一个故事灵感，希望生成完整的大纲文档。

## 工作流程

1. **读取背景资料**
   - 自动加载 romance-novel-skill 的资源文件：
     - `outline-method.md`（大纲创作方法论）
     - `templates/outline-template.md`（大纲格式模板）
     - `examples/outline-example.md`（优秀大纲示例）

2. **与用户互动**
   - **如果用户没有提供具体灵感**，主动询问关键信息：
     - 故事的核心主题或灵感是什么？
     - 男主角和女主角的职业、性格特点？
     - 两人相遇的方式？
     - 主要的情感冲突点？
     - 你希望的结局类型？

   - **如果用户已有具体想法**（例如"我想写一个医生和设计师的爱情故事"）：
     - 基于这个想法深入挖掘细节
     - 完善人设、冲突、情节等要素

3. **大纲创作**
   - 严格遵循 templates/outline-template.md 的格式
   - 按照大纲方法论的五章结构设计
   - 确保每章都有明确的情节线和情感变化
   - 避免平铺直叙，每个章节设置悬念或情节点

4. **输出与保存**
   - 将生成的完整大纲保存到项目根目录的 `outline.md` 文件
   - 主动告知用户大纲已创建，并简要介绍核心情节
   - 询问用户是否需要调整大纲，或者继续创作人物小传

## 输出格式

生成的 `outline.md` 文件将包含：
- 故事基本信息（标题、主题、类型）
- 核心设定（主要冲突、情感主线、故事背景）
- 详细的人物设定（女主、男主、配角）
- 五章详细的章节规划（每章的主题、关键情节、情感变化）
- 核心要素（情感爆点、金句、象征物）

## 注意事项

- 大纲一旦确定，将为后续所有章节创作提供基础，务必确保逻辑通顺
- 每章的关键情节要具体，避免空泛的描述
- 情感变化要清晰，确保人物行为符合性格设定
- 核心冲突要贯穿全文，避免前后脱节

## 下一步

大纲创作完成后，通常会继续：
1. 创作人物小传（使用 `/character` 命令）
2. 或直接进行章节目录规划（使用 `/catalog` 命令）
