# write - 撰写小说章节

**命令说明**：根据章节目录，撰写指定章节的完整正文。这是整个创作流程的最后一步，将大纲和人物设定转化为生动的故事文本。

**使用格式**：
- `/write 1` - 撰写第1章
- `/write 2` - 撰写第2章
- `/write 3` - 撰写第3章
- `/write 4` - 撰写第4章
- `/write 5` - 撰写第5章

**使用场景**：大纲、人物、目录都完成后，开始正式撰写小说章节。

## 前置检查

执行本命令前，必须确保所有前序文件存在：
- `outline.md`（故事大纲）
- `characters.md`（人物小传）
- `catalog.md`（章节目录）

如果任一文件缺失，提示用户先完成前序步骤。

同时需要检查目标章节是否已存在：
- `episodes/Episode-01.md`（第1章）
- `episodes/Episode-02.md`（第2章）
- `episodes/Episode-03.md`（第3章）
- `episodes/Episode-04.md`（第4章）
- `episodes/Episode-05.md`（第5章）

如果目标章节已存在，询问用户是覆盖重写还是继续写下一章。

## 工作流程

1. **读取背景资料**
   - 读取 `outline.md`（了解整体故事和核心冲突）
   - 读取 `characters.md`（了解人物性格和行为逻辑）
   - 读取 `catalog.md`（获取本章的情节规划）
   - 加载 romance-novel-skill 的资源文件：
     - `output-style.md`（写作风格指南）
     - `templates/chapter-template.md`（章节格式）
     - `examples/chapter-example.md`（优秀章节范例）

2. **章节创作**

   针对指定章节进行详细创作：

   - **第一章创作重点**：
     - 快速进入场景，一秒钟抓住读者
     - 生动地建立男女主角的人设对比
     - 制造有趣或充满火花的初次相遇
     - 结尾必须留下悬念或让读者产生好奇心

   - **第二章创作重点**：
     - 通过具体事件展现两人性格的闪光点
     - 增加互动频率，让关系自然升温
     - 埋下第三章冲突的种子
     - 用细节让情感萌芽不显得突兀

   - **第三章创作重点**：
     - 前半部分要足够甜蜜，让读者投入情感
     - 后半部分的冲突要有冲击力，制造情感落差
     - 冲突必须合理，源于人物性格和过往经历
     - 结局要痛苦但要让读者明白"还有希望"

   - **第四章创作重点**：
     - 真实地展现分开后的思念和痛苦
     - 通过回忆、第三方视角等方式让读者理解双方的内心
     - 展现人物的成长和觉悟
     - 为和解做充分的铺垫

   - **第五章创作重点**：
     - 和好过程要充分，不能仓促
     - 真诚对话要打动人心
     - 结局要圆满但不俗套
     - 呼应开篇细节，形成完整闭环

3. **写作技巧应用**
   - 遵循 `output-style.md` 中的所有写作规范
   - 运用五感描写增强代入感
   - 对话要推动情节或展现性格
   - 节奏控制好，快慢结合
   - 结尾留有悬念或情感冲击

4. **输出与保存**
   - 将章节正文保存到 `episodes/Episode-0X.md`
   - 字数控制在 2000-3000 字
   - 检查是否符合 `templates/chapter-template.md` 的结构
   - 询问用户是否满意，是否需要修改

## 输出格式

生成的章节文件包含：
- 约 2000-3000 字的完整正文
- 遵循现代言情风格、符合人物设定的对话和情节
- 严格按大纲和目录推进的剧情
- 包含检查清单和金句提炼功能

## 创作要点

- **严格遵循前序文档**：
  - 人物性格和行为必须符合 `characters.md` 的设定
  - 情节必须按 `catalog.md` 的规划展开
  - 核心冲突和主题必须呼应 `outline.md`

- **写作质量要求**：
  - 对话自然，符合口语习惯
  - 细节生动，有画面感
  - 情感真实，能引起共鸣
  - 节奏张弛有度
  - 避免过度抒情和空洞的描写

- **一致性检查**：
  - 人物设定不能前后矛盾
  - 情节逻辑要自洽
  - 情感变化要有铺垫
  - 伏笔要有呼应

## 注意事项

- 每个章节是一次性生成的完整初稿
- 生成后可以要求修改，但最好有明确的修改意见
- 章节顺序最好按1-2-3-4-5的顺序递进，便于保持情感连贯
- 如果用户想要调整剧情或人物，需要回到前序文档进行修改
- 每写完一章，建议稍作休息，确保下一章的质量
