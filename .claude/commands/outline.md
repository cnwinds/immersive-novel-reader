# outline

**描述**: 重新生成或修改故事大纲(会询问是否覆盖现有大纲)

**用法**: `/outline`

**参数**: 无

---

## 执行流程

当用户调用 `/outline` 命令时:

1. **检查现有大纲**
   - 如果 `outline.md` 文件已存在,询问用户是否覆盖
   - 用户确认后继续,否则中止

2. **读取技能包资源**
   - 读取 `skills/doomsday-skill/doomsday-outline-method.md` 获取创作方法论
   - 读取 `skills/doomsday-skill/templates/doomsday-outline-template.md` 获取格式规范
   - 读取 `skills/doomsday-skill/examples/outline-example.md` 学习优秀范例

3. **收集用户创意** (如果尚未收集)
   - Q1: 天灾类型和核心设定
   - Q2: 主角前世经历和重生动机
   - Q3: 主要反派和复仇对象

4. **生成完整大纲**
   
   大纲必须包含以下部分:
   
   - **故事梗概** (100-150字)
   - **三幕式结构**
     - 第一幕 (第1-20集): 重生准备+天灾爆发
     - 第二幕 (第21-45集): 生存竞争+复仇进行
     - 第三幕 (第46-60集): 终极对决+新秩序
   - **天灾进程体系**
     - 天灾类型和触发机制
     - 等级划分 (低级→中级→高级)
     - 时间线规划 (60集内的天灾升级节奏)
     - 视觉呈现 (适合漫画的画面描述)
   - **金手指设定**
     - 核心能力 (空间/预知/系统等)
     - 能力边界 (限制条件)
     - 成长路径 (能力如何升级)
     - 使用规则 (消耗/冷却/代价)
   - **核心冲突**
     - 外部冲突: 主角vs天灾、主角vs反派
     - 内部冲突: 主角心理成长线
   - **爽点分布规划**
     - 囤货爽点: 在60集中的分布节奏
     - 复仇爽点: 炮灰反派的打脸时间表
     - 高潮设计: 每20集一个大高潮

5. **写入文件**
   - 严格按照 `doomsday-outline-template.md` 的格式
   - 将大纲写入项目根目录的 `outline.md` 文件
   - 确保格式清晰、层次分明

6. **完成提示**
   ```
   ✅ 大纲已完成并保存到 outline.md
   
   📋 大纲包含:
   - 故事梗概
   - 三幕式结构 (60集布局)
   - 天灾进程体系
   - 金手指设定
   - 核心冲突设计
   - 爽点分布规划
   
   💡 下一步: 输入 /character 创建人物小传
   ```

## 注意事项

- **固定60集**: 不支持自定义集数,严格遵循60集结构
- **三幕对应**: 第1-20集(第一幕)、第21-45集(第二幕)、第46-60集(第三幕)
- **天灾升级**: 天灾等级必须从低到高渐进,不能突然跳级
- **爽点节奏**: 囤货爽点和复仇爽点要交错分布,避免集中在同一时段
- **视觉化**: 所有设定都要考虑漫画呈现的可行性

```

```
