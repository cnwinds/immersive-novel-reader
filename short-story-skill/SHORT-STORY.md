# 短篇世情文创作助手

## 系统介绍

这是一个专门用于创作短篇世情文（职场、情感、家庭、社会现实）的skill包。它将你的创意转化为完整的故事，采用五幕式结构，总计约10000字。

## 核心特点

✓ **简单直接**：四步完成创作（需求确认→大纲→分幕规划→正文）
✓ **结构清晰**：五幕式框架，层层递进，冲突升级
✓ **实用为王**：口语化写作、细节描写、对话推动，可操作性强
✓ **节奏掌控**：每幕2000字，万字左右完成一个完整故事

## 创作流程四步法

### 第一步：需求确认（/short-story-idea）

**触发条件**：用户输入 `/short-story-idea`

**执行流程**：
1. 询问用户一句话创意：你想写什么故事？（职场冲突/情感纠葛/家庭矛盾）
2. 询问叙事视角选择：
   - 第一人称（"我"视角，强代入感）
   - 第三人称全知（多线剧情）
   - 第三人称受限（聚焦主角）
3. 基于用户回答生成《故事核心创意》文档
4. 提示下一步："创意已明确，输入 /short-story-outline 构建大纲"

**产出文档**：`idea.md`

---

### 第二步：故事大纲（/short-story-outline）

**触发条件**：用户输入 `/short-story-outline`

**执行流程**：
1. 读取 `idea.md` 内容
2. 调用 short-story-skill，参考methodology和outline-template
3. 构建五幕式完整框架：
   - 第一幕：拉仇恨（2000字）
   - 第二幕：矛盾爆发（2000字）
   - 第三幕：持续施压（2000字）
   - 第四幕：高潮对抗（2000字）
   - 第五幕：结局与升华（2000字）
4. 每一幕明确核心目标、关键场景
5. 产出《五幕故事大纲》
6. 提示下一步："大纲已完成，输入 /short-story-plan-act 1 开始第一幕规划"

**产出文档**：`outline.md`

---

### 第三步：分幕规划（/short-story-plan-act）

**触发条件**：用户输入 `/short-story-plan-act <幕次>`

**执行流程**：
1. 读取 `outline.md` 获取该幕核心目标
2. 调用 short-story-skill，参考act-planning-template
3. 生成该幕详细规划：
   - 幕内场景（2-3个）
   - 场景的开头/发展/结尾
   - 角色状态变化（主角、反派、配角）
   - 情感体验设计（主打情绪）
   - 关键对话设计（对话双方、场景、功能）
   - 细节描写重点（视觉、听觉、触觉）
4. 产出《第X幕分幕规划》
5. 提示下一步："第X幕规划完成，输入 /short-story-write-act <幕次> 开始创作"

**产出文档**：`plan-act-X.md` （如：plan-act-1.md）

---

### 第四步：正文创作（/short-story-write-act）

**触发条件**：用户输入 `/short-story-write-act <幕次>`

**执行流程**：
1. 读取以下文档：
   - `idea.md`（故事核心创意）
   - `outline.md`（五幕大纲）
   - `plan-act-X.md`（本幕规划）
2. 调用 short-story-skill，遵循写作方法论
3. **分两次输出**（保证质量和字数）：
   - **第一次**：输出1000字+（开头 + 发展）
   - **中途指令**：用户输入"继续"，完成剩余内容（高潮 + 结尾）
4. 确保内容符合以下要求：
   - ✓ 口语化表达（像聊天，不文绉绉）
   - ✓ 细节描写充分（视觉、听觉、触觉）
   - ✓ 对话自然流畅（符合人物身份）
   - ✓ 行为展现内心（行胜于言）
   - ✓ 节奏快速紧凑（场景切换快）
5. 检查字数是否达到2000字左右
6. 产出《第X幕正文》
7. 提示下一步：
   - "第X幕完成！是否继续第X+1幕？输入 /short-story-plan-act X+1 继续"
   - 或："全部五幕完成！故事创作结束"

**产出文档**：`act-X.md` （如：act-1.md）

## Command 命令列表

### /short-story-idea
生成故事核心创意文档
- 通过问答明确创意和叙事视角
- 产出：idea.md

### /short-story-outline
生成五幕故事大纲
- 基于idea.md构建完整框架
- 产出：outline.md

### /short-story-plan-act <1-5>
生成指定幕次的分幕规划
- 参数：幕次（1-5）
- 读取：outline.md
- 产出：plan-act-{num}.md

### /short-story-write-act <1-5>
创作指定幕次的正文
- 参数：幕次（1-5）
- 读取：idea.md, outline.md, plan-act-{num}.md
- 产出：act-{num}.md
- **分两次生成**

### /short-story-continue
在 /write-act 中途继续输出剩余内容
- 用于承接第一次1000字+，完成2000字幕内容

### /short-story-status
查看当前创作进度
- 显示已完成的阶段和文档
- 提示下一步操作建议

## 写作规范与质量要求

### 语言风格要求
✓ **口语化写作**
- 不说"他感到十分愤怒" → 说"他气得直发抖"
- 不说"她心情沮丧" → 说"她整个人蔫了，话都不想说"
- 用短句，不用复杂句式

✓ **细节为王**
- 视觉：不说"办公室很乱" → 说"桌上堆满外卖盒，咖啡渍在桌上画圈"
- 听觉：不说"很吵" → 说"打印机嗡嗡声从九点就没停过"
- 触觉/感官：不说"很热" → 说"衬衫黏在后背，空调坏了"

✓ **对话推动故事**
- 对话符合人物身份（外卖员不说"此计甚妙"）
- 对话要短，3-5句一个回合
- 通过对话展现性格，不是大段对白

✓ **行为展现内心**
- 不说"他很紧张" → 写"他不停地看手机，手指在桌上敲"
- 不说"她很开心" → 写"她嘴角压不住，连着发错三次消息"

✓ **节奏要快**
- 一个场景不超过500字
- 对话和行动交替，不要大段描写
- 删掉冗余词汇

### 每幕质量控制清单

在提交每幕内容前，检查：

- [ ] 字数达到2000字左右
- [ ] 语言口语化，避免书面语和文艺腔
- [ ] 包含至少3-5个细节描写（看、听、触、闻）
- [ ] 对话自然，符合人物身份和场景
- [ ] 通过行为而非心理描写展现内心
- [ ] 节奏快，无冗余内容
- [ ] 与前期规划（plan-act-X.md）保持一致
- [ ] 开头第一句就抓住人
- [ ] 结尾留下钩子（引出下一幕）

## 创作原则与注意事项

### 核心原则

1. **四步流程不跳步**：必须按顺序完成 idea → outline → plan → write
2. **每幕都要规划**：写正文前必须有 plan-act-X.md
3. **分两次写**：每幕必须分两次输出，保证质量
4. **口语化优先**：念出来听着顺耳，才是好文字
5. **细节真实感**：真实的细节能让故事活过来

### 常见问题回避

❌ **避免的问题**：
- 事后诸葛亮式说教（"后来我才明白..."）
- 大段心理描写却缺乏行动
- 反转机械降神，毫无铺垫
- 人物脸谱化，非黑即白
- 为了虐而虐，脱离现实

✅ **推荐的做法**：
- 让读者自己感受人物情绪
- 通过对话和行为展现内心变化
- 反转的线索在前期有迹可循
- 反派有动机，行为符合底层逻辑
- 冲突真实可验证

## 故障排查与常见问题

### 问题1：/write-act 输出时与规划不符
**解决方法**：
- 重新读取 plan-act-X.md
- 严格按照规划中的场景和对话要点执行
- 如有偏离，请修正后重新输出

### 问题2：字数不够2000字
**解决方法**：
- 检查细节描写是否足够（增加感官细节）
- 对话是否充分展开（2-3段关键对话）
- 行为表现是否具体（增加动作和反应）
- 场景描写是否生动（环境、氛围、小物件）

### 问题3：语言太书面化
**解决方法**：
- 通读一遍，把书面语改成口语
- 用"的、了、呢、吧"等语气词
- 短句化：把长句拆成2-3个短句
- 模拟聊天语气
