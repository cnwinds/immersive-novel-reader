# character - 创作人物小传

**命令说明**：根据大纲文档，为故事中的主要角色（女主角、男主角及重要配角）创作详细的角色设定文档。

**使用场景**：大纲确定后，需要细化人物形象，为后续的章节创作提供人物行为依据。

## 前置检查

执行本命令前，必须确保：
- 项目根目录已存在 `outline.md` 文件
- 大纲中的人物基本信息已经确定

如果 `outline.md` 不存在，提示用户先使用 `/outline` 命令创建大纲。

## 工作流程

1. **读取背景资料**
   - 读取项目根目录的 `outline.md`（获取人物基础设定）
   - 加载 romance-novel-skill 的资源文件：
     - `templates/character-template.md`（角色小传格式）
     - `examples/character-example.md`（优秀角色小传示例）

2. **人物深度塑造**

   根据大纲提供的信息，分别为每个角色创作详细的人物小传，包括：

   - **女主角**：
     - 基本信息（姓名、年龄、职业、外貌特征）
     - 性格特质（优点、缺点、核心魅力）
     - 背景故事（家庭环境、教育经历、过往感情）
     - 核心驱动力（目标、需求、恐惧）
     - 人际关系网（家人、朋友、同事）
     - 独特的行为模式（口头禅、习惯性动作、生活细节）
     - 情感弧光（在故事中的成长变化）

   - **男主角**：
     - 基本信息与上述相同维度
     - 特别强调与女主角的对比和互补
     - 展现他的核心魅力和情感需求

   - **重要配角**（最多2-3个）：
     - 明确每个配角的功能（推动者、见证者、制造冲突者、情感支持者等）
     - 创建简明的角色设定（不必过于详细，但要有识别度）
     - 描述他们在故事中的关键作用

3. **人物关系图表**
   - 梳理人物之间的情感关系（爱情线、友情线）
   - 明确冲突关系（谁与谁有矛盾）
   - 构建人物对比表（展现男女主角的异同）

4. **输出与保存**
   - 将完整的人物小传保存到 `characters.md` 文件
   - 向用户确认人物小传已生成，并简要介绍主要角色的特色

## 输出格式

生成的 `characters.md` 文件包含：
- 女主角完整小传（按模板格式）
- 男主角完整小传（按模板格式）
- 重要配角简要设定（2-3个）
- 人物关系图与对比表
- 创作提示（对话风格、行为逻辑等）

## 创作要点

- **符合大纲**：所有角色设定必须基于大纲的基本信息，不能偏离
- **性格鲜明**：男女主角要有明显对比和互补性，不能同质化
- **行为逻辑**：每个角色的行为必须符合其性格和背景设定
- **记忆点**：为每个主要角色设计独特的"标签"（口头禅、习惯性动作等），增强辨识度
- **功能清晰**：配角必须有明确的功能，不能只是为了填满故事

## 注意事项

- 主角的行为逻辑要贯穿全文，后续章节必须遵循这些设定
- 配角的戏份要适当，不能抢主角风头
- 人物的衣着、习惯等细节要符合当代都市背景
- 避免脸谱化，即使是配角也要有真实感

## 下一步

人物小传完成后，通常会继续：
1. 创作章节目录（使用 `/catalog` 命令）
2. 或直接开始撰写具体章节（使用 `/write` 命令）
